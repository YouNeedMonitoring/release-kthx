# Agent Instructions

## Commit Messages

- Use Conventional Commits for every commit.
- Format: `type(scope): summary`
- Allowed `type` values: `feat`, `fix`, `chore`, `docs`, `refactor`, `test`, `ci`, `build`, `perf`.
- Keep `summary` imperative and concise.

Examples:

- `feat(release): add publish-on-merge mode`
- `fix(github): improve pull request permission error`
